RescueSession

The connection between your application and the Rescue Technician Console is represented by a RescueSession object. This is a singleton object you access it by calling the sharedInstance class method.

Configure session
  • apiKey

    Unique identifier for the app.

    Every app need an API key. Starting a Rescue session will always fail without it.

  • appGroup

    App group identifier.

    Using the broadcast feature the app and the extension have to communicate with eachother. The communication is working over group user defaults container. Set the app group identifier in your entitlements (both for app and extension) and here.

  • delegate

    The object that acts as the delegate of RescueSession.

  • connectToSessionAutomatically

    Pass this parameter along with session start.The SDK will automatically join the session if it is true. If it is set to false the SDK will notify the RescueSessionDelegate, that the session is ready with rescueSessionReady(). After the session ready call, you can call connectSession() on the RescueSession to connect he session; or you can connect to the sesion from your broadcast extension.

  • sessionStartParameters

    This property stores the session start configuration.

    Automatically created, call its methods to configure it.

Session information
  • sessionStatus

    State of the session.

  • operatorName

    Name of the technician.

    This property is KVO compliant

  • sessionStart

    Date-time of the start of the session.

    This property is KVO compliant

  • transmittedData

    Transmitted data during session

  • afterSessionSurveyURL

    URL of the customer survey.

    The URL can change during a support session, so only get this property after ending the support session.

  • broadcastDelegate

Control session lifecycle
  • startSession()

    Start a Rescue session using the configuration from the sessionStartParameters object.

    The startSession method is asynchronous, it will return immediately. You will be notified via the delegate about the outcome of the session start.

  • connectSession()

    Connect to the Rescue session

    Important: If you initiated the Rescue session with connectToSessionAutomatically set to false, you have to manually call connectSession() or broadcastSession()

  • broadcastSession()

    Prepare the Rescue session for broadcasting.

    Important: If you initiated the Rescue session with connectToSessionAutomatically set to false, you have to manually call connectSession() or broadcastSession()

  • endSession()

    End the Rescue session.

    The endSession method is asynchronous, it will return immediately. You will be notified via the delegate about the outcome of the session start.

  • reset()

    Clean up the Rescue SDK, delete every data stored by it.
